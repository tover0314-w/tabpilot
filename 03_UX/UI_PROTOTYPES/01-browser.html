chrome.sidePanel

chrome.sidePanel MV3 Stable since Chrome 114

Use the chrome.sidePanel API to host content in the browser's side panel — a UI surface that lives alongside the web page, persistent across navigations, and available without the user opening a popup.

Concepts

The side panel sits to the right of the page on desktop layouts. It is a per-extension region: a single extension owns it, but the user can pin it open, close it, or change the side it appears on (left or right).

Unlike chrome.action popups, the side panel does not dismiss when the user clicks outside it. This makes it suitable for ongoing conversations, persistent tools, and — as in our case — a long-running agent.

Manifest

"side_panel": {
  "default_path": "sidepanel.html"
},
"permissions": ["sidePanel", "tabs", "tabGroups"]

Open programmatically

chrome.sidePanel.open({ tabId: tab.id })
  .then(() => console.log('opened'))

The promise resolves once the panel has been displayed. Errors occur if the user has not enabled the extension in this tab, or if the panel is pinned closed by the user (configurable from chrome://extensions).
