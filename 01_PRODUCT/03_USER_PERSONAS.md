# 用户画像

## Persona 1：办公知识工作者 Olivia

### 背景

Olivia 是典型办公和知识工作者。她每天会打开邮箱、日历、Google Docs/Notion、Slack/Teams、表格、任务系统、会议资料、内部后台和客户/项目相关页面。

### 行为

- 每天打开 30–80 个 tabs，常常分布在多个 Chrome 窗口。
- 经常重复打开同一个文档、报表、会议链接或客户页面。
- 会使用 Chrome tab groups，但觉得手动分类麻烦。
- 愿意尝试能减少重复整理工作的 AI 工具。

### 痛点

- 顶部 tab bar 全是 favicon，看不到标题。
- 想把沟通、文档、报表、项目、客户资料分开，但不愿手动整理。
- 多个窗口之间任务混杂，切换时容易丢上下文。
- 需要快速判断哪些页面值得保留、哪些是重复打开。

### 价值点

- 一键整理所有普通窗口。
- 邮件/文档/会议/报表/项目自动归类。
- 当前页面总结。
- Workspace 保存和恢复。

## Persona 2：独立开发者 Alex

### 背景

Alex 是独立开发者，正在做一个 SaaS 产品。每天会打开 GitHub、Vercel、Supabase、Stripe、Linear、Notion、文档和竞品页面。

### 行为

- 每天打开 40–100 个 tabs。
- 经常重复打开同一个 GitHub issue 或文档页面。
- 会使用 Chrome tab groups，但觉得手动分类麻烦。
- 愿意尝试 AI 工具和 Chrome 扩展。

### 痛点

- 顶部 tab bar 全是 favicon，看不到标题。
- 想把 PR、文档、调试、竞品分开，但不愿手动整理。
- 需要快速理解 GitHub repo 或技术文档。
- 需要恢复昨天的开发上下文。

### 价值点

- 一键整理 GitHub/Docs/DevTools。
- GitHub PR 自动归到 Code Review。
- 当前页面总结。
- Workspace 保存和恢复。

## Persona 3：产品经理 Mia

### 背景

Mia 正在写 PRD，同时打开竞品、用户反馈、数据仪表盘、Figma、Notion、Linear、文档和相关文章。

### 行为

- 同时研究多个 feature。
- 需要从多个页面提炼需求和产品洞察。
- 经常保留 tabs 作为“稍后参考”。

### 痛点

- 竞品研究 tabs 混在需求文档和设计稿里。
- 很多 tabs 只是灵感来源，但最后忘了看。
- 手动整理会打断写作心流。

### 价值点

- 按项目/任务自动分类。
- Group summary 总结竞品页面。
- Dashboard 保存 PRD workspace。
- Multi-tab chat 提炼需求和对比表。

## Persona 4：研究型学生 / 研究员 Sam

### 背景

Sam 做论文或行业研究，会打开论文、搜索结果、博客、视频和工具页面。

### 行为

- 每个研究主题打开大量资料页。
- 想知道哪些文章值得读。
- 会把 tabs 当成临时阅读列表。

### 痛点

- 无法快速判断哪些资料有用。
- 重复打开搜索结果和论文页面。
- 不知道如何保存研究上下文。

### 价值点

- Research group 自动分类。
- 当前页面摘要。
- Group summary。
- Workspace 导出为 markdown。

## Persona 5：咨询/投资分析师 Rachel

### 背景

Rachel 做行业研究，经常打开公司官网、融资新闻、LinkedIn、报告、竞品和数据源。

### 行为

- 需要快速比较多个公司和产品。
- 需要保存研究轨迹。
- 对隐私和数据控制更敏感。

### 痛点

- 资料来源分散。
- 手动整理和总结耗时。
- 很多页面有敏感客户/公司信息。

### 价值点

- 多 tab 对比表。
- Workspace 归档。
- 隐私等级控制。
- 自配 API key / local mode。

## 首发建议

首发优先服务 Persona 1：办公知识工作者，同时保留独立开发者作为早期传播和反馈人群。

原因：

- 办公多 tab 是更大众、更高频的真实问题。
- SEO 关键词可以覆盖更广泛的 Chrome productivity / tab organizer 搜索需求。
- 独立开发者仍适合 Product Hunt、HN、Reddit、X 的早期传播。
- 两类用户共享同一个 aha：点击一次，所有窗口里的标签页变清楚。
